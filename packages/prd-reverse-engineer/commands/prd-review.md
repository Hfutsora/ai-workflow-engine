# PRD 审查 — 质量检查

审查已生成的 PRD 文档质量，检查模板完整性、跨文档一致性和内容准确性。

## 执行

使用 `prd-reviewer` agent（`.claude/agents/prd/prd-reviewer.md`）执行审查。

遵循 `.claude/skills/prd/reverse-engineer.md` 中的质量标准和评审标准（第十、十一章）。

## 审查要点

1. **模板完整性**：章节结构是否完整、每个章节是否有实质内容
2. **产品视角一致性**：是否始终使用业务语言，无技术术语泄露
3. **需求完整性**：背景、场景、功能说明、业务规则是否齐全
4. **5.x 子分类**：是否包含 5.1 已完成需求、5.2 待做需求、5.3 已完成优化及补充需求
5. **跨文档一致性**：术语、交叉引用、模块边界
6. **代码抽样验证**：抽查功能描述与源代码的一致性

## 输出

`prd/review-report.md` — 包含总览评分、各 PRD 评分、问题清单、跨文档一致性分析。

## 后续步骤

- `/prd-fix` 根据审查报告自动修复问题
