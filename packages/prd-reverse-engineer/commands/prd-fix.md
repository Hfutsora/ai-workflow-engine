# PRD 修复 — 根据审查报告自动修复

读取 `prd/review-report.md`，按优先级自动修复已识别的问题。

## 执行流程

### Step 1: 读取审查报告

读取 `prd/review-report.md`，解析问题清单，按严重级别分组：
- **严重**：必须修正（缺少章节、编造功能、技术语言泄露）
- **高**：应当修正（章节缺失、命名不一致、结构不规范）
- **中**：建议修正（轻微技术用词、格式问题）
- **低**：可选修复（风格不一致、拼写）

### Step 2: 排除已失效的问题

检查每个问题是否仍然存在（可能已在之前的修复中解决）：
- 用 Grep 验证问题文本是否仍在文件中
- 跳过已修复的问题

### Step 3: 并行修复

对独立的 PRD 文件启动并行 Agent 修复，每个 Agent：
1. 读取对应 PRD 文件
2. 遵循 `.claude/skills/prd/reverse-engineer.md` 的全部标准
3. 修复该文件中的所有问题
4. 使用 Edit 工具精确修改，不影响其他内容

修复类型对应处理方式：

| 问题类型 | 修复方式 |
|----------|----------|
| 技术语言泄露 | 将技术术语改写为业务语言 |
| 5.x 子分类缺失 | 添加空章节骨架（"暂无"） |
| 章节顺序错误 | 调整标题编号和内容位置 |
| 章节命名不一致 | 统一为模板标准名称 |
| 术语不一致 | 使用 replace_all 统一术语 |

### Step 4: 验证

修复完成后，对每个修改过的文件进行快速验证：
- 确认技术术语已消除
- 确认章节结构完整
- 确认未引入新问题

### Step 5: 更新审查报告

在 `prd/review-report.md` 末尾追加修复记录：

```markdown
## 修复记录

> 修复日期: YYYY-MM-DD

| 问题编号 | 状态 | 修复说明 |
|----------|------|----------|
| C-01 | 已修复 | ... |
| H-02 | 已修复 | ... |
```

## 注意事项

- 修复时只改问题本身，不做额外重构
- 对不确定的内容标注"待确认"，不擅自编造
- 如果某个问题无法自动修复（如需要业务确认），标记为"需人工处理"并说明原因
